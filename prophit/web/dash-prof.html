Java -prof

Previous : Profiler Types   Up : Profiler Types   Next : hprof

-prof is a profiler that originally shipped with the JDK 1.1. Using -prof is very simple, you just run your Java program with the -prof option:

	java -prof -classpath <path> your.Test

Advantages

  • Built into the JDK
  • Very simple to run
  • Works with JDKs 1.1 and above
  • Generally does not require the JIT compiler (or HotSpot) to be disabled in order to function properly
  • Fine-grained timing information seems to be quite accurate for single-threded programs (see below about multi-threaded programs)

Disadvantages

  • Does not work with multi-threaded programs. Profile data for all the threads is all mixed up together
  • Has a significant performance impact. You can expect your program to take several times longer to run than without the -prof option. The HelloList program takes 2:31 to profile with -prof and 0:28 to profile with hprof, both running JDK 1.3.
  • Only captures 2 levels of stack depth. If function A calls function X which calls function Y, and function B calls function X which calls function Y, -prof does not provide any way to tell how much time in function Y was in calls from A and how much was in calls from B.
  • Not very much information is available about -prof

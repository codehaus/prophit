HelloList java -prof Profile

Open this sample using the File/Sample Profiles/prof/HelloList menu item.

This sample shows java -prof output from the sample program HelloList.java. This sample profile may be generated using the default settings of the dashprof Ant task.

Explore the HelloList hprof Profile before moving on to this one.

  1. The first thing you may notice about this profile is that the 'time' numbers are all larger. For example, the inclusive time of <root> is 87,027 in this profile, where as it was 9,671 for the hprof profile. The -prof profile also took a lot longer to generate! The cpu=samples mode of hprof is faster, but gathers less data, than cpu=times or -prof.
  2. The hprof data agrees very well with the -prof data. arrayAndString and linkedAndString take about 40% of the time spent in HelloList.main each, whereas arrayAndBuffer and linkedAndBuffer only take about 10% of the time each. The functions which use ArrayList are slightly faster than the ones which use LinkedList.
